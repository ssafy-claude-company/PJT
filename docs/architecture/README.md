# Organt Core — 아키텍처 문서

> Discord 위에서 사는 AI 직원(**Organt**)들이 단일흐름 '베턴'으로 협업하는 시스템의 Core(`src/`) 아키텍처 정리.
> **목적**: 추후 리팩토링의 토대. 현재 구조를 정확히 도식·서술하고, 패턴/컨벤션의 좋은 점·나쁜 점·체계가 필요한 부분을 **근거(file:line)와 함께** 표시한다.

이 문서 세트는 코드를 **수정하지 않는다**. 평가·표시는 별도 문서에 모으고, 모든 주장은 `ref/`에 file:line으로 근거를 단다.

## 표기 규약

- **근거**: 본문의 `src/파일.py:line`은 클릭 가능한 위치 참조다. 굵은 주장에는 `[R-NN]` 근거 번호를 달고 `ref/REFERENCES.md`에서 코드 인용·논리를 본다.
- **평가 라벨**:
  - ✅ **좋음(good)** — 유지·확산할 가치가 있는 패턴.
  - ⚠️ **개선(bad)** — 안티패턴·버그유발·결합·중복 등 고쳐야 할 것.
  - 🏗️ **체계필요(needs-system)** — 지금은 동작하나 체계(추상화·계약·타입·에러 분류 등)가 없어 흔들리는 것.
- **검증**: 각 평가는 워크플로우의 적대적 검증(인용 정확성 + 주장 방어가능성)을 거쳤다. 검증 요약은 06 문서 말미와 PR 본문에 있다.

## 범위 / 비범위

- **범위**: 루트 `src/`(Organt Core) — 13개 프로덕션 모듈(~8.5K LOC) + `tests/`·`scripts/`·`organt/`(시드·인격) + 루트 문서 정합성.
- **비범위**: `organt_sns/`(별도 SNS 웹앱 — 자체 `docs/` 보유). 이번 정리는 Core에 한정한다.

## 방법론(요약)

1. 제어흐름 스파인(`sys_core`·`communication`·`protocol`·`organt`·`main`)을 전수 정독해 1차 근거 확보.
2. 13개 모듈을 병렬 분석(워크플로우) → 제어흐름·의존성·패턴 종합.
3. 모든 발견의 `file:line` 인용을 소스에 대조해 **적대적 검증**(인용 정확성 + 주장 타당성).
4. 검증된 근거로만 문서·도식 작성, 근거는 `ref/`로 분리.
