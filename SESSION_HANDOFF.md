19. **경쟁 구현(compete)은 도입 직후 사용자 판단으로 제거**: "같은 지능 2개 비교해봤자 거기서
   거기 — 효과는 협업에서 나온다." 기술적으로 타당: 전 봇이 같은 파운데이션 모델이라 독립
   구현이라도 실패 모드가 상관됨(N-버전 다양성 가정 불성립). LLM의 실질 다양성은 재샘플링이
   아니라 **역할(직군 기준·경험)이 다른 상호작용**에서 나온다. 중요 작업의 다인 협업은 기존
   기제가 담당: owner 구현 + 같은 직군 교차 리뷰(request(Info)) + QA(Work) + (fork로 싸진)
   회의·표결. 남긴 것: ① 쓰기 리스 훅(permissions 2.5 + `flow.write_lease`) — **휴면 인프라**
   (병렬 Work/alive-집합 도입 시 재사용, 리스 비면 비용 0, 테스트 유지) ② `flow.fork_kind`
   게이트 — 회의·표결 가지(프레임 없음)의 선구현 차단에 **현역 필수** ③ `_fork_collect`의
   kind 인자(휴면). **일반 Work 병렬(QA 중첩·파이프라이닝)은 의도적 보류** — 가지에 중첩 요청
   권리가 필요해 alive-집합 개편(대대적)이 전제. 발동 조건: 프로젝트 병렬로도 처리량 부족
   또는 흐름 내 대기가 실측 병목일 때.
20. **fork 동시성 가드(`flow.fork_active`) — 재감사에서 발견한 실구멍 봉쇄**: fork 중엔 베턴
   (alive)이 리더에 머물러, CLI가 같은 턴에 병렬 도구 호출(vote+request, vote+meet)을 내면
   수집 가지와 같은 동료를 이중으로 깨워 '같은 봇 두 턴'(같은 스코프 세션 파일 충돌)이 될 수
   있었다(직렬 vote 시절엔 alive 이동이 자연 차단하던 것). 수집 중 신규 request/vote/meet는
   '[대기]'(조인 보장으로 짧음), 조인 후 즉시 해제. meet 2라운드+에서 멤버가 라운드 사이에
   타 흐름에 점유되면 회의 전체 break가 아니라 그 멤버만 continue(부분 진행).

### 직군 '권한(Discord 역할)' 모델 — 운영 메모
- 직군 = 권한 없는 순수 라벨 역할(assign_job_role이 생성). '관리자' 등 위험 권한 보유 역할과
  managed(봇 통합) 역할은 직군이 아니며 시스템이 절대 건드리지 않는다(`_is_job_role` 판정).
- assign_job_role은 '라벨과 동기화': 라벨(겸직이면 '주직군·부직군')의 역할을 모두 보장하고,
  라벨에 없는 직군 역할 잔재는 제거한다 — 누적 스택 재발 방지.
- 진실원 우선순위: 런타임 jobs.json > Discord 역할(리클레임 복원용) > .env 로스터 라벨.
- 리더(담당자)는 직군이 아니다 — To 지정으로 흐름마다 바뀌며 역할 변경을 일으키지 않는다.

## 자격증명 / 실행 (중요 — 실제로 어떻게 도는가)
- **영속 환경변수**(컨테이너 바뀌어도 유지)에는 봇 토큰이 4개만 있다:
  `SYSTEM_BOT`(라우터) + 워커용 `TEST_BOT_1`·`TEST_OBT_2`·`TEST_OBT_3`.
  → 이대로 `run_listener.sh`를 돌리면 **워커 3명(프론트엔드·디자이너·예비) 기준으로 정상 동작**한다.
  (`CHANNEL_ID`·`DEPLOY_NAME`은 `run_listener.sh`가 기본값을 주입하므로 별도 설정 불필요.)
- **풀팀(워커 ~20명)**을 돌리려면 `ORGANT_BOT_2~20` 토큰이 필요하다. 이 토큰들은 영속 env에
  등록된 적이 없고 과거 `.env`에만 있었다(현재 없음). 풀팀 운영 = 이 토큰들을
  **환경설정(웹 UI)의 환경변수에 등록**해야 컨테이너 교체에도 유지된다.
  로스터(`run_listener.sh`)는 "토큰이 있으면 그만큼, 없으면 건너뜀"으로 자동 적응한다 —
  토큰을 더 등록할수록 워커가 더 붙는다(코드 변경 불필요).
  (2026-06-10: 사용자가 토큰 20개 전부를 이 컨테이너 `.env`에 제공 — **휘발**이므로 리클레임 후엔
  다시 받거나 웹 UI 환경변수로 영속화해야 한다. 배포 키(GH_PAT 등)도 같은 상태.)
- **LLM 인증**: `ANTHROPIC_API_KEY`는 env에 없다. 별도 호스트에서 돌리려면 주입 필요.
